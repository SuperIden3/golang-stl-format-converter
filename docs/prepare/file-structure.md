# File Structure

A standard Go project file structure combined with early documentation.

Early project documentation, as shown the `docs/` directory, is produced because, as a growing engineer in trade, documentation is just as important. Plus, to get more attached to *design briefs* and *decision matrices*.

```txt
golang-stl-format-converter
├── cmd/
│   ├── api/
│   │   └── main.go
│   └── gui/
│       └── main.go
├── docs/
│   └── prepare/
│       └── file-structure.md
├── go.mod
├── internal/
│   ├── api/
│   │   └── handlers.go
│   └── converter/
│       └── converter.go
├── LICENSE
├── pkg/
└── README.md
```

> For this project, there will only be a design brief. This is because this poses as an individual solution and cannot be used in a comparison of solutions, as the purpose of the decision matrix.
